iDOC= policy

  1. Docs only related to Commodore eight-bit computers. We accept docs from other platforms as long as they parallel stuff that was made for Commodore eight-bit computers. If the only version of the docs that are available are for non-Commodore machines, then those can be entered here, but will be removed when the originals for Commodore computers are available.
  2. The primary format for iDOC= documentation is raw text, which means that graphics need to be converted to ascii-art or descriptions to convey information or maybe even feelings. If images are needed, other formats for publication may be necessary, please contact Mike Naberezny at mike@naberezny.com for more information.
  3. Preferably non-English documents only. Since there already is the original Project 64 for English docs, I see no reason to include them here as well. Some English translations of the docs might be included, though, and English documents for machines not covered by Project 64.
  4. "Hacker docs" are tolerated. Nth generation etexts of unknown origin are included within the project but are likely to replaced with etexts of known originals when available.
  5. iDOC= does not distribute games or other software, just documentation. Most of the software mentioned here probably exists on somebody's web page or at Arnold or somewhere else.
  6. iDOC= honours the original copyrights of the included documentations. Since most Commodore eight-bit stuff is no longer manufactured or sold, we don't believe that there is any clash of interests in this free distribution of documents. If, however, you have something to object to this distribution, please contact Mike Naberezny at mike@naberezny.com, so that we can sort it out. If you are the copyright holder of a featured documentation and do not want it distributed here, we will of course withdraw it as soon as possible.
  7. iDOC= is a non-profit project.
